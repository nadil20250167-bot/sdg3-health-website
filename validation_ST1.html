Reflection on the Validation Process

While validating my pages, I found a CSS selector mistake on this Validation Page: I wrote .ss img instead of .ss, so my screenshot sizing rules never applied and one image stretched the whole page sideways. I also found an unclosed ul tag and a duplicate "Student Details" section in my footer, plus a missing heading inside a gallery article and a skipped heading level (h2 to h4) in the footer. I fixed each by re-reading the code line by line rather than relying on how the page looked in the browser, since invalid HTML doesn't always look broken visually. The main lesson was that the validator catches structural mistakes the browser silently ignores, so checking early and often is more reliable than only checking at the end.
